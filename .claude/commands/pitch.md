# /pitch

Verkaufs-Präsentation für die fertige Seite (`$ARGUMENTS` = Kunde/Projekt).

## Voraussetzung
Seite ist durch `/ship` (fehlerfrei, Build grün) und hat einen **Vercel-Preview-Link**.

## Ablauf (Denke wie in `/marketing`)
1. **Aufhänger** — der eine Satz, warum diese Seite den Kunden weiterbringt
   (mehr Buchungen/Anfragen/Premium-Wahrnehmung). Kein Feature-Dump.
2. **Vorher → Nachher** — was war das Problem, was löst die neue Seite konkret.
3. **Live zeigen** — Preview-Link, 2–3 Wow-Momente (Animation, Speed, Mobile).
4. **Wert > Preis** — Nutzen zuerst, dann Preisrahmen (Paket-Optionen statt einer Zahl).
5. **Klarer nächster Schritt** — eine eindeutige CTA (Termin, Zusage, Go-Live-Datum).

## Output
- Kurzer Pitch-Text (sprechfertig, kein Corporate-Jargon).
- Optional: Case-Study-Entwurf fürs eigene Portfolio → Input für `/marketing` (nächster Kunde).

## Regel
Verkauft wird Ergebnis, nicht Technik. Der Kunde kauft mehr Umsatz/Status — nicht GSAP.
